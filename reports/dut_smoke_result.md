## Implementation Details

### 1. Mock DUT Model

**File:** `tests/mock_dut.py`

Simulates the Python model that would be generated by Picker, providing:
- Toffee-like async API (`wait_cycles`, signal `read`, signal `write`)
- Complete Cache behavior (read/write, hit/miss, line fill, LRU eviction, dirty writeback)
- Valid/ready handshake protocol
- CPU-side and Memory-side interfaces

### 2. Signal Map Configuration

**File:** `configs/signal_map.yaml`

Maps abstract signal names to concrete DUT signal names:
- CPU request/response interface
- Memory request/response interface

### 3. Toffee Cache Adapter

**File:** `src/cache_vip/toffee_adapter.py`

Implements the adapter that translates between:
- DUT-independent verification core (`CacheTxn`, `CacheResponse`)
- Concrete DUT signals driven/sampled via Toffee/Picker

Implemented methods:
- `reset()`: Assert/deassert reset signal
- `drive_cpu_request()`: Drive CPU request following valid/ready protocol
- `sample_cpu_response()`: Sample CPU response from DUT
